# Ethical Hacking Task 01

## Information Gathering & Reconnaissance

## Objective

The objective of this task is to understand the reconnaissance phase of ethical hacking by collecting publicly available information about a target website using passive information gathering techniques.

## Target Selection

**Website Name:** Amazon

**Website URL:** https://www.amazon.com/

**Reason:**

* Publicly accessible website
* Safe for passive reconnaissance
* Used for educational purposes only

## Tools Used

1. Kali Linux
2. whois
3. dig
4. host
5. nslookup
6. curl
7. OpenSSL

## Commands Used

### 1. WHOIS Lookup

```bash
whois amazon.com
```

**Purpose:**
Displays domain registration information such as registrar, creation date, expiry date, name servers, and domain status.

### 2. DNS Lookup

```bash
dig amazon.com
```

**Purpose:**
Retrieves DNS records for the target domain.

### 3. Name Server Lookup

```bash
dig NS amazon.com
```

**Purpose:**
Displays the authoritative name servers of the domain.

### 4. Mail Server Lookup

```bash
dig MX amazon.com
```

**Purpose:**
Finds the mail exchange servers used by the domain.

### 5. Host Lookup

```bash
host amazon.com
```

**Purpose:**
Resolves the domain name to its IP address and displays DNS information.

### 6. IP Address Lookup

```bash
nslookup amazon.com
```

**Purpose:**
Finds the IP address associated with the domain.

### 7. HTTP Header Information

```bash
curl -I https://www.amazon.com
```

**Purpose:**
Retrieves HTTP response headers from the web server.

### 8. SSL Certificate Information

```bash
echo | openssl s_client -connect amazon.com:443 | openssl x509 -noout -dates -issuer -subject
```

**Purpose:**
Displays SSL certificate issuer, validity period, and subject details.

## Information Collected

* Domain Name
* Registrar
* Registration Date
* Expiry Date
* Name Servers
* Domain Status
* IP Address
* HTTP Headers
* SSL Certificate Details

## Observation

Passive reconnaissance provides useful information about a publicly accessible website without interacting directly with its internal systems or attempting exploitation.

## Conclusion

This exercise demonstrated the importance of information gathering during the initial phase of a penetration test. Using standard Kali Linux tools, publicly available information about a domain can be collected safely and ethically.
